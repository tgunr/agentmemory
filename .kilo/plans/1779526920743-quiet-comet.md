# Plan: Kilo Session Manager for AgentMemory

## Goal

Enable users to browse Kilo sessions (both Local and Cloud tabs from the history UI), preview their content, and selectively save them to AgentMemory as sessions with extractable memories.

## Architecture Overview

```
User ──→ MCP Tools ──→ REST Endpoints ──→ Kilo Session Functions ──→ AgentMemory Session/Observation/Memory Storage
          (Browse)        (Import)            (Parse + Extract)
```

## Components

### 1. New MCP Tools (4 tools)

| Tool | Purpose | Parameters |
|------|---------|------------|
| `kilo_sessions_list` | List Kilo local/cloud sessions | `source: "local" \| "cloud"`, `limit?: number`, `workspace?: string` |
| `kilo_session_preview` | Preview a specific session's content | `sessionId: string`, `source: "local" \| "cloud"` |
| `kilo_session_save` | Save a Kilo session to AgentMemory | `sessionId: string`, `source: "local" \| "cloud"`, `options?: {saveObservations: boolean, saveMemories: boolean, memoryTypes?: string[]}` |
| `kilo_session_memories` | Extract specific parts of a session as memories | `sessionId: string`, `source: "local" \| "cloud"`, `extractWhat: "decisions" \| "patterns" \| "bugs" \| "architecture" \| "all"` |

### 2. New REST Endpoints (6 endpoints)

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/agentmemory/kilo-sessions/local` | GET | List local Kilo sessions |
| `/agentmemory/kilo-sessions/cloud` | GET | List cloud Kilo sessions |
| `/agentmemory/kilo-sessions/preview` | POST | Get full session data for preview |
| `/agentmemory/kilo-sessions/import` | POST | Import a session to AgentMemory |
| `/agentmemory/kilo-sessions/extract` | POST | Extract parts as memories |
| `/agentmemory/kilo-sessions/batch` | POST | Batch import multiple sessions |

### 3. New Functions (4 functions)

| Function | Purpose |
|----------|---------|
| `mem::kilo-list-local` | Parse `~/.kilocode/cli/global/global-state.json` and return session list |
| `mem::kilo-list-cloud` | Use Kilo SDK to fetch cloud sessions via `/kilo/cloud-sessions` API |
| `mem::kilo-preview` | Load and parse session data (metadata + messages + conversation history) |
| `mem::kilo-import` | Create AgentMemory session with observations, summary, and optional memories |
| `mem::kilo-extract` | Analyze session content and extract specific memory types |

### 4. Session Data Flow

#### Local Sessions
1. Read from `~/.kilocode/cli/global/global-state.json` (session index)
2. Load session details from `~/.kilocode/cli/global/tasks/{sessionId}/`
3. Parse `task_metadata.json`, `ui_messages.json`, `api_conversation_history.json`
4. Extract: first prompt, file edits, tool usage, decisions, errors
5. Create AgentMemory session with parsed data

#### Cloud Sessions  
1. Use Kilo SDK's `cloudSessions()` method or direct API call
2. Fetch session list from `/kilo/cloud-sessions`
3. For preview/import: fetch from `ingest.kilosessions.ai/api/session/{id}/export`
4. Parse cloud session format (info + messages + parts)
5. Create AgentMemory session with parsed data

### 5. Memory Extraction Strategy

When saving a session, the system can create:

| Memory Type | Source | Content |
|-------------|--------|---------|
| **Session Summary** | Entire session | Title, narrative, key decisions, files, concepts |
| **Decision Memories** | Tool calls with file edits | What was decided, why, files changed |
| **Pattern Memories** | Repeated operations | Workflow patterns discovered in the session |
| **Bug Memories** | Error messages + fixes | Problems encountered and solutions |
| **Architecture Memories** | File structure changes | System design decisions and rationale |
| **Observation Records** | All tool usage | Compressed observations for future recall |

### 7. Authentication & Security

- Cloud session access requires `kilocodeToken` from `~/.kilocode/cli/config.json`
- REST endpoints use existing AgentMemory auth (Bearer token)
- MCP tools use existing MCP router permissions
- Never expose auth tokens in responses or logs

### 8. Error Handling

- Handle missing Kilo installation gracefully
- Handle cloud API failures with clear messages  
- Validate session data before import
- Provide progress feedback for long imports
- Support retry for failed imports

### 9. User Experience Flow

```
1. User: "Show me my Kilo sessions"
2. Agent: Lists recent sessions (local + cloud) with titles, dates, workspaces
3. User: "Preview session abc-123"
4. Agent: Shows session summary, key interactions, files modified
5. User: "Save this session"
6. Agent: Creates AgentMemory session with:
   - Session record with metadata
   - Compressed observations for tool usage
   - Session summary
   - Optional: individual memories for decisions/patterns/bugs
7. User: "What memories were extracted?"
8. Agent: Lists created memories with types and content
```
